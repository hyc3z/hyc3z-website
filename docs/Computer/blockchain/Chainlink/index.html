Shared Data Resource​

Data的来源，是由Community维护的，有点消息队列的味道。有Consumer，有Aggregator，当然中间还有Proxy。

点开了看了下，有很多

消息是由多个Oracle发出到链上经Aggregator整合后产生的，当然时效性肯定没有交易所的K线图那么实时，毕竟区块确认也需要时间。目前设置的Trigger Parameter是变动超过0.5%，或者一小时。

当然，行情数据没有那么实时，但是天气数据等等还是相当不错的。

看到这里是不是也有点手痒痒？因为从这个Chain上，我们也可以Consume数据，获取到去中心化的信息。
